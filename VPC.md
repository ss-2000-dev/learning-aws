# VPC 作成手順

#### VPC 作成

VPC Lambda を試したい

1. VPC を作成 → 「VPC など」を選択
2. IPv4 CIDR ブロック: 10.0.0.0/16
3. IPv6 CIDR ブロックなし
4. テナンシー: デフォルト
5. AZ の数: 2
6. パブリックサブネットの数: 2
   - ap-northeast-1a のパブリックサブネット CIDR ブロック: 10.0.0.0/20
   - ap-northeast-1c のパブリックサブネット CIDR ブロック: 10.0.16.0/20
7. プライベートサブネットの数: 2
   - ap-northeast-1a のプライベートサブネット CIDR ブロック: 10.0.128.0/20
   - ap-northeast-1c のプライベートサブネット CIDR ブロック: 10.0.144.0/20
8. NAT ゲートウェイ: なし
9. VPC エンドポイント: なし
10. DNS オプション
    - DNS ホスト名を有効化
    - DNS 解決を有効化

# セキュリティグループ (Security Group) は、基本的にホワイトリスト方式

- 明示的に許可されたトラフィックのみ通過
- インバウンド（EC2 インスタンスなどへの受信）も、アウトバウンド（EC2 インスタンスなどからの送信）も、ルールで許可された IP アドレス、ポート、プロトコルのみが許可され、それ以外はすべて暗黙的に拒否されます。
